\section{Objective}
The objective of this assignment is to demonstrate the concept of paging in operating systems.
The program simulates logical address translation using three page table schemes:

\begin{itemize}[leftmargin=1.2cm]
    \item Single-level page table
    \item Two-level page table
    \item Hashed page table
\end{itemize}

The address size is assumed to be 8 bits. Each logical address is translated into a physical address, and then the program accesses data from simulated physical memory.

\section{Address Format}
The logical address size is:

\[
m = 8 \text{ bits}
\]

The offset size is:

\[
d = 3 \text{ bits}
\]

Therefore, the page size is:

\[
2^d = 2^3 = 8 \text{ bytes}
\]

For the single-level and hashed page tables, the remaining 5 most significant bits represent the page number:

\[
p = 8 - 3 = 5 \text{ bits}
\]

\[
\text{Logical Address} = [\text{Page Number: 5 bits}][\text{Offset: 3 bits}]
\]

For the two-level page table, the page number is divided into two parts:

\[
p_1 = 2 \text{ bits}, \quad p_2 = 3 \text{ bits}
\]

\[
\text{Logical Address} = [p_1: 2 \text{ bits}][p_2: 3 \text{ bits}][d: 3 \text{ bits}]
\]

\section{Memory Simulation}
Physical memory is simulated as a byte array. The program uses 8 frames, and each frame contains 8 bytes.

\[
\text{Physical Memory Size} = 8 \times 8 = 64 \text{ bytes}
\]

After translation, the physical address is calculated using:

\[
\text{Physical Address} = (\text{Frame Number} \times \text{Page Size}) + \text{Offset}
\]

\section{Page Table Implementations}
\subsection{Single-Level Page Table}
The single-level page table is implemented as a simple array. The virtual page number is used directly as the index, and the value stored at that index is the frame number.

\begin{codebox}[Single-level address translation]
\begin{lstlisting}[style=cstyle]
page = logical_address >> 3
offset = logical_address & 7
frame = page_table[page]
physical_address = frame * 8 + offset
\end{lstlisting}
\end{codebox}

\subsection{Two-Level Page Table}
The two-level page table divides the page number into two parts. The first part selects the outer table, and the second part selects the entry inside the inner table.

\begin{codebox}[Two-level address translation]
\begin{lstlisting}[style=cstyle]
p1 = (logical_address >> 6) & 3
p2 = (logical_address >> 3) & 7
offset = logical_address & 7
frame = outer_table[p1][p2]
physical_address = frame * 8 + offset
\end{lstlisting}
\end{codebox}

In the implementation, the inner table is allocated only when it is needed. This shows the main idea of hierarchical paging: not every second-level table has to exist at the start.

\subsection{Hashed Page Table}
The hashed page table uses a simple hash function:

\[
\text{bucket} = \text{page number} \bmod 16
\]

The assignment allows assuming no collisions, so each used bucket stores one page number and its mapped frame number.

\begin{codebox}[Hashed page table lookup]
\begin{lstlisting}[style=cstyle]
page = logical_address >> 3
bucket = page % HASH_BUCKETS
frame = hash_table[bucket].frame
physical_address = frame * 8 + offset
\end{lstlisting}
\end{codebox}

\section{Conclusion}
This assignment shows how paging translates logical addresses into physical addresses. The single-level page table uses direct indexing, the two-level page table divides the page number into two indexes, and the hashed page table uses a hash bucket for lookup. In all three cases, the final physical address is formed by combining the mapped frame number with the offset.
