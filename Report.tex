\section{Introduction}

When physical memory is exhausted, allocating threads can enter \textbf{direct reclaim}: they block while the kernel evicts pages synchronously. Under coordinated memory thrash, many threads reclaim in parallel, evicting pages that are immediately refaulted. The working set exceeds available RAM, swap I/O rises, and tail latency degrades.

\begin{figure}[H]
    \centering
    \begin{tikzpicture}[
        node distance=0.55cm,
        box/.style={rectangle, draw=aublue, rounded corners, thick, align=center,
            text width=5.8cm, inner sep=4pt, font=\footnotesize\sffamily},
        arr/.style={-{Stealth[length=2mm]}, thick, aublue!65}
    ]
        \node[box] (n1) {Many threads compete for physical RAM};
        \node[box, below=of n1] (n2) {RAM fills; allocations take the allocator slowpath};
        \node[box, below=of n2] (n3) {Many threads enter \textbf{direct reclaim} together};
        \node[box, below=of n3] (n4) {Synchronous reclaim contends; refault and swap traffic spike};
        \draw[arr] (n1) -- (n2);
        \draw[arr] (n2) -- (n3);
        \draw[arr] (n3) -- (n4);
    \end{tikzpicture}
    \caption{Conceptual chain from contention to reclaim storms (motivation for coordinating direct reclaim).}
    \label{fig:thrash-flow}
\end{figure}

\textbf{PressurePause} is a kernel modification that adds a bounded, PSI-aware coordination step on the direct reclaim path. When memory PSI \texttt{some} avg10 exceeds 0.01\% (basis-point compare against the same fixed-point values as \texttt{/proc/pressure/memory}), the patch wakes \texttt{kswapd} explicitly and applies a short, capped wait for background progress, then always runs the stock \texttt{shrink\_zones()} priority loop. The design never disables reclaim permanently and always preserves stock behavior on timeout or when guards fire.

\begin{figure}[H]
    \centering
    \includegraphics[width=\linewidth,height=0.42\textheight,keepaspectratio]{problem-heavy-memory.png}
    \caption{Host under heavy memory pressure during a memory-intensive workload (elevated swap use and reclaim-related activity). This motivates coordinating foreground direct reclaim with background reclaim.}
\end{figure}

This report covers the development environment and baseline build (Sections~2--3), component identification (Section~4), design and patch implementation (Sections~5--6), correctness properties (Section~7), and evaluation evidence that the new code path runs (Section~8).

				gfp_t gfp_mask, nodemask_t *nodemask)

	struct scan_control sc = {
		.nr_to_reclaim = SWAP_CLUSTER_MAX,
		.gfp_mask = current_gfp_context(gfp_mask),

	BUILD_BUG_ON(MAX_PAGE_ORDER >= S8_MAX);
	BUILD_BUG_ON(DEF_PRIORITY > S8_MAX);
	BUILD_BUG_ON(MAX_NR_ZONES > S8_MAX);

	if (throttle_direct_reclaim(sc.gfp_mask, zonelist, nodemask))

	return nr_reclaimed;

\subsection{Watermarks}

Reclaim is triggered when a zone's free page count falls below configured \textbf{min}, \textbf{low}, or \textbf{high} watermarks during allocation. The fast path in \texttt{get\_page\_from\_freelist()} fails; \texttt{\_\_alloc\_pages\_slowpath()} may wake \texttt{kswapd}, attempt compaction, and finally call \texttt{\_\_alloc\_pages\_direct\_reclaim()} so the allocating thread performs synchronous reclaim.

\subsection{kswapd vs.\ Direct Reclaim}

Both paths converge on \texttt{shrink\_node()}, but they differ in caller context and entry points:

\subsection{Pressure Stall Information (PSI)}

PSI accounts time tasks spend stalled on memory, I/O, or CPU. For PressurePause, the \textbf{gate} uses memory PSI \texttt{some} avg10 (tasks stalled on memory); \texttt{full} is also sampled for debug peaks.

\begin{itemize}[leftmargin=1.2cm]
    \item \textbf{Types:} \texttt{PSI\_MEM\_SOME}, \texttt{PSI\_MEM\_FULL} in \srcfile{include/linux/psi_types.h} (unchanged by the patch).
    \item \textbf{Accounting / readers:} \srcfile{kernel/sched/psi.c} extends existing PSI averaging with \texttt{psi\_mem\_full\_avg10()} and \texttt{psi\_mem\_some\_avg10()} (\srcfile{include/linux/psi.h} declares them).
    \item \textbf{Userspace:} \texttt{/proc/pressure/memory} (\texttt{some} vs.\ \texttt{full} averages)
\end{itemize}

\texttt{psi\_memstall\_enter()} in \texttt{\_\_alloc\_pages\_direct\_reclaim} marks the \emph{current allocating task} as memory-stalled during reclaim for PSI statistics. That is distinct from \texttt{psi\_mem\_some\_avg10()}, which the patch reads at the hook to gate bounded coordination before \texttt{shrink\_zones()}.

\subsection{Hook Justification}

\texttt{shrink\_node()} is shared by both \texttt{kswapd} and direct reclaim. Hooking only at \texttt{do\_try\_to\_free\_pages()} keeps the change on the allocating-thread path. Hooking at \texttt{shrink\_node()} would require extra guards and would affect a broader surface.

Build and boot screenshots for the baseline kernel appear in Section~3; line numbers above refer to the patched v6.8.12 tree in \srcfile{kernel/linux}.

\section{Design and Alternative Methodologies}

\textbf{Chosen approach:} PSI-gated bounded backoff on direct reclaim entry---gate on memory PSI \texttt{some} avg10, wake \texttt{kswapd}, wait up to \texttt{PRESSURE\_PAUSE\_MAX\_MS}, then fall through to stock reclaim on timeout or lack of progress.

static unsigned long psi_mem_avg10(enum psi_states state)
{
	unsigned long avg;
	u64 now;

	if (static_branch_likely(&psi_disabled))
		return 0;

	mutex_lock(&psi_system.avgs_lock);
	now = sched_clock();
	collect_percpu_times(&psi_system, PSI_AVGS, NULL);
	if (now >= psi_system.avg_next_update)
		psi_system.avg_next_update = update_averages(&psi_system, now);
	avg = psi_system.avg[state][0];
	mutex_unlock(&psi_system.avgs_lock);

	return avg;
}

/* True if any managed zone in the zonelist still needs reclaim (not at min). */
static bool pressure_pause_under_pressure(struct zonelist *zonelist,
					  struct scan_control *sc)
{
	struct zoneref *z;
	struct zone *zone;
	bool any = false;

	for_each_zone_zonelist_nodemask(zone, z, zonelist, sc->reclaim_idx,
					sc->nodemask) {
		if (!managed_zone(zone))
			continue;
		any = true;
		if (!zone_watermark_ok(zone, sc->order, min_wmark_pages(zone),
				       sc->reclaim_idx, 0))
			return true;
	}
	return false;
}

/*
 * PSI-gated bounded coordination before synchronous shrink_zones().
 * Falls through to stock reclaim on timeout or lack of progress.
 */
static void pressure_pause_maybe(struct zonelist *zonelist,
				 struct scan_control *sc)
{
	unsigned long psi_full, psi_some;
	unsigned long deadline;

	atomic64_inc(&pressure_pause_entered);

	if (cgroup_reclaim(sc)) {
		atomic64_inc(&pressure_pause_skip_cgroup);
		return;
	}

	psi_full = psi_mem_full_avg10();
	psi_some = psi_mem_some_avg10();
	pressure_pause_note_psi(psi_some, psi_full);

	/* Gate on memory PSI "some" -- rises earlier than "full" under swap thrash. */
	if (!psi_mem_avg10_exceeds_bps(psi_some, PSI_MEM_THRESHOLD_BPS)) {
		atomic64_inc(&pressure_pause_skip_psi);
		return;
	}

	if (!pressure_pause_has_reclaimable(zonelist, sc)) {
		atomic64_inc(&pressure_pause_skip_no_reclaimable);
		return;
	}

	/* Only skip if no zone in the zonelist is under min watermarks. */
	if (!pressure_pause_under_pressure(zonelist, sc)) {
		atomic64_inc(&pressure_pause_skip_zones_ok);
		return;
	}

	pressure_pause_wake_kswapd(zonelist, sc);
	atomic64_inc(&pressure_pause_activations);

	deadline = jiffies + msecs_to_jiffies(PRESSURE_PAUSE_MAX_MS);
	while (time_before(jiffies, deadline)) {
		if (fatal_signal_pending(current))
			break;
		if (pressure_pause_any_zone_ok(zonelist, sc))
			break;
		schedule_timeout_idle(1);
	}
}

\subsection{\texttt{pressure\_pause\_maybe()} control flow}

On each \texttt{do\_try\_to\_free\_pages()} entry the hook increments \texttt{entered}, then:
\begin{enumerate}[leftmargin=1.2cm]
    \item \textbf{Cgroup reclaim} $\rightarrow$ increment \texttt{skip\_cgroup}, return (stock reclaim only).
    \item Read \texttt{psi\_mem\_some\_avg10()} and \texttt{psi\_mem\_full\_avg10()}; update \texttt{peak\_psi\_*} maxima.
    \item If \texttt{some} avg10 is below 0.01\% (\texttt{PSI\_MEM\_THRESHOLD\_BPS=1}) $\rightarrow$ \texttt{skip\_psi\_low}, return.
    \item If no zone has reclaimable pages $\rightarrow$ \texttt{skip\_no\_reclaimable}, return.
    \item If \emph{no} zone in the zonelist is below min watermarks $\rightarrow$ \texttt{skip\_zones\_ok}, return. (Stock logic incorrectly skipped when \emph{any} zone was OK; the fix requires at least one zone still under pressure.)
    \item Wake \texttt{kswapd} per managed zone; increment \texttt{activations}; bounded wait up to \texttt{PRESSURE\_PAUSE\_MAX\_MS} (20\,ms), exiting early on fatal signal or if any zone reaches min watermarks.
    \item Always fall through: the stock \texttt{shrink\_zones()} priority loop runs unchanged.
\end{enumerate}

\subsection{PSI threshold change}

The first implementation used \texttt{LOAD\_INT(psi\_full) >= 25}, which requires \textbf{25\%} memory \texttt{full} stall (the integer part of the fixed-point average matches the whole number in \texttt{/proc/pressure/memory}, e.g.\ \texttt{avg10=25.00}). Benchmarks peaked near \texttt{full avg10=0.18} (0.18\%), so the gate never fired and \texttt{activations} stayed at zero.

\section{Correctness and Safety}

\begin{enumerate}[leftmargin=1.2cm]
    \item \textbf{Bounded wait:} the pause is capped at 20\,ms and uses one-tick idle sleep slices.
    \item \textbf{Never skip reclaim:} the hook always returns to the stock \texttt{shrink\_zones()} priority loop.
    \item \textbf{kswapd cooperation:} \texttt{wakeup\_kswapd()} is additive; foreground reclaim still proceeds.
    \item \textbf{Livelock guard:} skip pause if \texttt{zone\_reclaimable\_pages()} is zero for all zones in the zonelist.
    \item \textbf{Fallback:} Ubuntu \texttt{6.8.0-117-generic} and \texttt{6.8.12-baseline} remain in GRUB if the patched kernel panics.
\end{enumerate}

\subsection{Discussion}

Kernels built with the original \texttt{LOAD\_INT(psi\_full) >= 25} gate could complete stress workloads with \textbf{zero} \texttt{activations} because observed \texttt{full} avg10 never reached 25\%. After the basis-point gate on \texttt{some} avg10 (Section~6.3), debugfs confirms the hook runs: on the 8\,vm / 93\% workload, \texttt{activations} rose from 15 to 55 (+40) during the run; on one 4\,vm / 85\% trial (\texttt{160617}), +15 activations. Matched baseline vs patched runs also show lower swap-out and residual swap on the patched kernel, with higher \texttt{pgmajfault} as a documented tradeoff (average swap-out $-$17.9\% on 8\,vm / 93\%, $-$31.7\% on the 4\,vm / 85\% patched average).

\subsection{Limitations}

This is not a full statistical study. PSI samples are every 5\,s and can miss short spikes seen at the reclaim hook, which is why the debugfs activation counter is more important than sampled \texttt{avg10} alone. Baseline PSI still reports ``no PSI,'' so baseline and patched PSI totals cannot be compared directly. Major-fault counts rise on the patched kernel; future work should add repeated trials, latency percentiles, and a workload sweep to find where the swap reduction is worth the fault-cost tradeoff.

\section{Conclusion}

PressurePause adds \texttt{pressure\_pause\_maybe()} on the direct-reclaim path: when memory PSI \texttt{some} avg10 exceeds 0.01\%, the allocating thread wakes \texttt{kswapd}, waits up to 20\,ms, then always proceeds with stock \texttt{shrink\_zones()}. Supporting changes export system PSI averages, fix the zone watermark guard, and expose debugfs counters for verification. Under \texttt{stress-ng}, the patched kernel is stable and debugfs shows real activations after the threshold fix; matched benchmarks report lower swap pressure at the cost of higher major page faults. The project delivers a small, PSI-gated reclaim coordination patch with observable behavior and measured tradeoffs.
